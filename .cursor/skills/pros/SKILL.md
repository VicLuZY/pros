# PROS Skill

Minimal workflow scaffold for `/pros`.

## When to use
- User runs `/pros`
- User asks to initialize lightweight project workflow files

## Behavior
1. Check for `.pros/README.md` and `.pros/notes.md`.
2. Create missing files with concise starter content.
3. Do not overwrite existing user-authored content.
4. Keep output minimal and practical.

## Starter content

### `.pros/README.md`
- Title: `# PROS`
- One-line purpose statement.
- A short "Next steps" checklist with 3 items.

### `.pros/notes.md`
- Title: `# PROS Notes`
- Date section.
- Bulleted scratchpad area.
